# 简历制作与查看网站（前后端不分离 · Python）项目方案

## 1. 项目目标
构建一个**前后端不分离**的简历网站，支持：
- 在线创建/编辑简历
- 预览与导出（PDF/打印友好）
- 多份简历管理
- 登录注册与权限控制
- 模板切换与基础样式美化

> 适用场景：个人自用、校招投递、团队内部简历统一管理。

---

## 2. 技术选型（前后端不分离）
- 主框架：**Flask** 或 **Django**
- 模板引擎：**Jinja2**(Flask) / **Django Template Language**
- 数据库：**SQLite**（开发阶段）/ **MySQL/PostgreSQL**（生产）
- ORM：**SQLAlchemy**（Flask）或 **Django ORM**
- 表单校验：**WTForms** / Django Forms
- 认证：Session 认证（可扩展 JWT）
- PDF 生成：**WeasyPrint / pdfkit**
- 前端：Bootstrap 5 + 少量 JS（非前后端分离）

---

## 3. 推荐方案与权衡

### 方案 A（推荐）：Flask 轻量实现
- 优点：结构清晰、学习成本低、灵活
- 缺点：插件需自行选型整合
- 适合：个人/小项目快速落地

### 方案 B：Django 一体化
- 优点：开箱即用（Admin/Auth/ORM）
- 缺点：框架较重、定制成本略高
- 适合：需要后台管理和复杂权限的项目

### 方案 C：Flask + Jinja + HTMX 增强交互
- 优点：不引入 SPA 也能提升体验
- 缺点：需额外维护 HTMX 组件
- 适合：想要“不分离”但交互更顺畅

**建议**：先采用方案 A（Flask），后续按需升级方案 C。

---

## 4. 系统架构（MVC）
`
Browser
  -> Flask路由 (View)
  -> Service/Model (Controller/Model)
  -> SQLite/MySQL (DB)
  -> 返回 HTML 模板
`

核心模块：
- 用户模块：注册/登录/退出
- 简历模块：新建/编辑/删除/复制
- 模板模块：切换样式/预览
- 导出模块：生成 PDF/打印版
- 配置模块：站点信息、邮箱（可选）

---

## 5. 数据模型（初版）
- User(id, username, email, password_hash, created_at)
- Resume(id, user_id, title, template_id, visibility, updated_at)
- ResumeSection(id, resume_id, type, title, content_json, sort_order)
- Template(id, name, description, preview_img, css_path)

> content_json 采用 JSON 存储灵活字段（经历、项目、技能等）。

---

## 6. 页面与路由设计
- /：首页
- /auth/login：登录
- /auth/register：注册
- /resume/list：我的简历
- /resume/create：新建简历
- /resume/<id>/edit：编辑简历
- /resume/<id>/preview：预览
- /resume/<id>/export：导出 PDF
- /settings：个人设置

---

## 7. MVP 范围（第一阶段）
1. 用户注册登录
2. 简历创建与编辑（基础字段）
3. 在线预览
4. PDF 导出
5. 简历列表管理

---

## 8. 开发里程碑
- M1：项目骨架 + 用户系统
- M2：简历编辑与预览
- M3：模板系统与 PDF 导出
- M4：美化、测试与部署文档

---

## 9. 风险与应对
- PDF 样式差异：固定模板 + 字体打包
- 数据一致性：编辑采用草稿/版本策略
- 安全性：密码哈希 + CSRF + 权限校验
- 可维护性：分层结构 + 配置外置
