# 简历制作与查看网站 · 细化方案（前后端不分离 · Python）

## 0. 背景与目标
基于现有《项目方案》，进一步细化为可直接开发落地的版本。  
默认技术路线建议：**Flask + Jinja2 + SQLAlchemy + Bootstrap5 + WeasyPrint**。

---

## 1. 最终选型建议（可执行版）
- 后端：**Flask 3.x**
- 模板：**Jinja2**
- ORM：**SQLAlchemy 2.x + Flask-SQLAlchemy**
- 表单：**Flask-WTF + WTForms**
- 认证：**Flask-Login**
- 数据库：开发 SQLite / 生产 MySQL 或 PostgreSQL
- 样式：Bootstrap 5
- PDF：WeasyPrint（优先）或 pdfkit（备选）
- 增强交互（可选）：HTMX（后期）

---

## 2. 数据库字段细化（ER 约定）

### 2.1 User
- id: int, pk
- username: str(32), unique
- email: str(254), unique
- password_hash: str(128)
- is_active: bool, default True
- created_at: datetime
- updated_at: datetime

### 2.2 Resume
- id: int, pk
- user_id: int, FK -> User.id
- title: str(120)
- target_position: str(120), nullable
- visibility: enum('private','public'), default 'private'
- template_id: int, FK -> Template.id
- is_default: bool, default False
- created_at: datetime
- updated_at: datetime

### 2.3 ResumeSection（模块化简历内容）
- id: int, pk
- resume_id: int, FK -> Resume.id
- section_type: enum('basic','education','experience','project','skill','award','summary','custom')
- title: str(120)
- content_json: JSON
- sort_order: int, default 0
- visible: bool, default True
- created_at / updated_at

> 建议每个 section_type 有 JSON schema 约束，避免脏数据。

#### content_json 示例（experience）
```json
{
  "company": "OpenAI",
  "role": "后端工程师",
  "start": "2024-06",
  "end": "2025-09",
  "highlights": ["搭建简历服务", "优化 PDF 导出性能"]
}
```

### 2.4 Template
- id: int, pk
- name: str(64), unique
- description: str(256)
- preview_img: str(512)
- css_path: str(512)
- is_active: bool, default True

### 2.5 ExportRecord（审计与复用）
- id: int, pk
- user_id: int
- resume_id: int
- file_path: str(1024)
- file_size_kb: int
- status: enum('success','failed')
- created_at: datetime

---

## 3. 表单与校验规则

### 3.1 注册
- username: 4~32，字母数字下划线
- email: 合法邮箱格式
- password: 8~64，至少包含大小写和数字
- confirm_password: 一致校验

### 3.2 登录
- email + password
- 连续失败限流：5 次/10 分钟（后期）

### 3.3 简历编辑
- title 必填
- 至少包含一个 section（basic + summary 或 experience）
- 时间区间 start <= end
- 高亮项数量限制（<=10 条/section）

---

## 4. 页面信息架构（IA）

### 4.1 全局布局
- 顶部导航：Logo、我的简历、模板市场、导出记录、退出
- 移动端：响应式折叠导航

### 4.2 页面清单
1. 首页 `/`
   - 价值点介绍 + CTA（开始制作）
2. 登录 `/auth/login`
3. 注册 `/auth/register`
4. 简历列表 `/resume/list`
   - 卡片视图：标题、目标岗位、更新时间、操作（编辑/预览/导出/删除）
5. 新建简历 `/resume/create`
6. 编辑器 `/resume/<id>/edit`
   - 左侧：section 列表与排序
   - 右侧：表单编辑区
   - 顶部：保存/预览/导出按钮
7. 预览 `/resume/<id>/preview`
   - 所见即所得模板渲染
8. 导出 `/resume/<id>/export`
   - 单按钮生成 PDF + 下载
9. 个人设置 `/settings`
   - 修改邮箱、密码

---

## 5. 交互流程（关键）

### 5.1 创建简历流程
1. 点击“新建简历”
2. 输入标题、目标岗位、选择模板
3. 自动生成默认 section 骨架（基础、教育、经历、技能）
4. 进入编辑器

### 5.2 编辑保存策略
- 手动保存 + 离开提醒（beforeunload）
- 后期可扩展自动保存（节流 2s）

### 5.3 导出流程
1. 点击导出
2. 后端取最新简历数据 + 模板
3. 服务端渲染 HTML -> PDF
4. 返回下载链接并记录 ExportRecord

---

## 6. 模板系统设计（核心）

### 6.1 目录约定
```
templates_resume/
  classic/
    preview.png
    style.css
  modern/
    preview.png
    style.css
```

### 6.2 渲染流程
- 预览：Jinja 模板 + 对应 CSS
- 导出：复用同一 HTML，注入 CSS 内联（提高 PDF 兼容）

### 6.3 样式约束
- A4 页面 210mm x 297mm
- 正文字号 10.5~11pt
- 页边距 12~16mm
- 打印分页：避免段落断裂（break-inside: avoid）

---

## 7. 权限与安全

- 所有简历接口需登录（Flask-Login）
- 仅 owner 可编辑/删除/导出
- 公开简历需要单独生成“只读分享链接”（可选）
- CSRF 防护（Flask-WTF 默认）
- 密码哈希（werkzeug/passlib）
- 日志脱敏：不记录密码、token

---

## 8. 错误处理与提示

- 表单错误：字段级提示 + 顶部汇总
- 404：简历不存在或无权限
- 500：返回统一错误页 + 日志 trace_id
- 导出失败：提示重试并记录 ExportRecord
